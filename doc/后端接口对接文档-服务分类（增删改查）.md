# 洗衣店后台管理系统 - 后端接口对接文档（服务卡片/服务分类/服务项：三表联动）

> 本文档用于前后端联调对齐。
>
> 业务背景：
> - 前端页面：`/#/service/category`
> - 主要管理对象：`home_service_card`
> - 关联表：`laundry_category`、`laundry_service`
>
> 核心要求：
> - 对 `home_service_card` 的新增/修改/删除必须**事务性联动**维护 `laundry_category`
> - 删除 `home_service_card` 时，必须同时删除该分类下的 `laundry_service`
>
> 前端已按“聚合接口方案（推荐）”实现对接：后端提供聚合 CRUD 接口，后端在单事务中完成三表一致性。

---

## 1. 基本约定

### 1.1 Base URL
- 管理端接口前缀：`/api/admin`

### 1.2 鉴权
- `Authorization: Bearer <token>`

### 1.3 统一响应结构（与前端 axios 封装一致）

前端 `src/utils/request.ts` 按如下结构解析：

```json
{
  "code": 0,
  "data": {},
  "msg": "ok"
}
```

- `code == 0`：成功，前端返回 `data`
- `code != 0`：失败，前端弹出 `msg`

### 1.4 时间格式
- `YYYY-MM-DD HH:mm:ss`

## 3. 三表联动规则（必须实现）

### 3.1 新增 home_service_card
后端在同一事务内：
- 新增 `laundry_category`（得到 `categoryId`）
- 新增 `home_service_card` 并写入 `category_id = categoryId`
- 若传入 `services`，批量新增 `laundry_service`（均使用 `category_id = categoryId`）

### 3.2 修改 home_service_card
后端在同一事务内：
- 更新 `home_service_card`
- 更新其关联的 `laundry_category`（通过 `category_id` 或请求体中的 `category.id`）
- 对 `laundry_service` 做增删改同步（建议：以 `services` 为准进行差量或全量替换）

### 3.3 删除 home_service_card
后端在同一事务内：
- 删除 `home_service_card`
- 删除其关联的 `laundry_category`
- 删除该分类下所有 `laundry_service`（`laundry_service.category_id = categoryId`）

> 建议：服务项数量通常不大，可采用“全量替换”的方式简化后端实现。

## 4. 接口清单（聚合接口：推荐方案）

- `GET    /api/admin/home-service-card` 服务卡片列表
- `GET    /api/admin/home-service-card/{id}` 服务卡片详情（含服务项）
- `POST   /api/admin/home-service-card` 新增（事务联动三表）
- `PUT    /api/admin/home-service-card/{id}` 修改（事务联动三表）
- `DELETE /api/admin/home-service-card/{id}` 删除（事务联动三表 + 级联删除服务项）

## 7. 新增服务卡片（事务联动）

## 9. 删除服务卡片（事务联动 + 级联删除服务项）

**接口地址：** `DELETE /api/admin/home-service-card/{id}`

### 9.1 功能说明
后端需在同一事务中完成：
- 删除 `home_service_card`
- 删除其对应的 `laundry_category`
- 删除该分类下全部 `laundry_service`

### 9.2 响应
- `data: null` 即可
