# 后端接口对接文档-系统设置（字典管理&参数配置&操作日志）

## 0. 文档目的
本文档用于后台管理系统“系统设置”模块对接，面向：
- 前端开发
- AI/自动化工具（生成页面/联调脚本）

模块包括：
- 字典管理（sys_dict / sys_dict_item）
- 参数配置（sys_config）
- 操作日志（sys_oper_log）

---

## 1. 通用约定

### 1.1 Base URL
- 后台接口统一前缀：`/api/admin`

### 1.2 鉴权
- 请求头携带：`Authorization: Bearer <token>`

### 1.3 通用响应结构（AdminResult）
```json
{
  "code": "00000",
  "data": {},
  "msg": "操作成功"
}
```

### 1.4 逻辑删除（deleted）
- 本模块所有新表均使用 MyBatis-Plus 逻辑删除字段：`deleted`
- 约定：
  - `deleted=0`：未删除
  - `deleted=1`：已删除
- 删除接口调用后端 `removeById(...)` 时，将自动转为 `UPDATE ... SET deleted=1`

### 1.5 常见错误码
- `A0300`：参数错误/资源不存在/不允许的操作

# 2. 字典管理（系统设置-字典）

## 2.1 字典分页列表
### GET `/api/admin/system/dict/page`

#### Query
- `page`：默认 1
- `size`：默认 10
- `keyword`：可选（匹配 dictCode/dictName）
- `status`：可选（1启用/0禁用）

# 3. 参数配置（系统设置-参数配置）

## 3.0 推荐配置项（已接入业务）

### 3.0.1 admin.order.maxQueryDays

- **用途**：限制后台【订单管理-订单列表】接口 `GET /api/admin/order` 在传入 `startDate`/`endDate` 时，允许查询的最大时间跨度（天）。
- **类型**：int
- **默认值**：90
- **生效方式**：无需重启；修改后立即生效。
- **触发条件**：仅当同时传 `startDate` 与 `endDate` 时生效。
- **超限行为**：返回 `A0300`，msg 类似：`查询时间跨度过大，最大允许 90 天`。

## 3.1 参数配置分页列表
### GET `/api/admin/system/config/page`

#### Query
- `page`：默认 1
- `size`：默认 10
- `keyword`：可选（匹配 configKey/remark）
- `status`：可选（1启用/0禁用）

---

## 3.2 新增参数配置
### POST `/api/admin/system/config`

#### Body
```json
{
  "configKey": "admin.order.maxQueryDays",
  "configValue": "90",
  "valueType": "int",
  "status": 1,
  "remark": "后台订单列表最大查询跨度（天）"
}
```

---

## 3.3 更新参数配置
### PUT `/api/admin/system/config/{id}`

#### Body
```json
{
  "configValue": "60",
  "valueType": "int",
  "status": 1,
  "remark": ""
}
```

说明：
- `configKey` 作为唯一键，默认不建议修改。

---

## 3.4 删除参数配置（逻辑删除）
### DELETE `/api/admin/system/config/{id}`

---

## 3.5 获取启用的配置值（调试/读取）
### GET `/api/admin/system/config/get?key=admin.order.maxQueryDays`

Response.data：
- 成功：字符串值，例如 `"90"`
- 不存在/禁用：`null`

# 4. 操作日志（系统设置-操作日志）

## 4.1 操作日志分页查询
### GET `/api/admin/system/log/page`

#### Query
- `page`：默认 1
- `size`：默认 10
- `startDate`：可选（YYYY-MM-DD）
- `endDate`：可选（YYYY-MM-DD）
- `userId`：可选
- `success`：可选（1成功/0失败）
- `keyword`：可选（匹配 path/username）

#### Response.data
```json
{
  "total": 10,
  "records": [
    {
      "id": 1,
      "userId": 4,
      "username": "管理员",
      "method": "GET",
      "path": "/api/admin/system/dict/page",
      "ip": "127.0.0.1",
      "userAgent": "PostmanRuntime/7.x",
      "requestParams": "page=1&size=10",
      "responseCode": "200",
      "success": 1,
      "costMs": 12,
      "createTime": "2026-03-16 17:10:00",
      "deleted": 0
    }
  ],
  "pages": 1
}
```

---

## 4.2 操作日志写入机制（后端实现说明）
- 由 `AdminOperateLogInterceptor` 自动记录：
  - 拦截 `/api/admin/**`
  - 排除 `/api/admin/auth/login`、`/api/admin/auth/captcha`
- 记录时机：请求完成后（afterCompletion）写入 `sys_oper_log`
- 当前记录的 requestParams：仅 queryString（不读取 body，避免流被消费）
